# Правила контенту презентації

## Структура презентації (35 хв)

| Секція | Час | Слайди | Опис |
|--------|-----|--------|------|
| 1. Вступ | 5 хв | 4 | Про спікера, що таке Agentic IDE |
| 2. Правила (Rules) | 10 хв | 7 | Типи, структура, приклади |
| 3. Команди (Commands) | 8 хв | 6 | Slash-команди, синтаксис |
| 4. Навички (Skills) | 8 хв | 6 | MCP, автоматизації |
| 5. Висновки | 4 хв | 4 | Поради, ресурси, Q&A |

## Дизайн слайдів

### Загальні правила
- Максимум **5 пунктів** на слайд
- Темна тема (developer-friendly)
- Чіткі, короткі заголовки
- Код важливіший за текст

### Типографіка
- Заголовки: стислі, без крапки
- Bullet points: починати з дієслова
- Код: монохромний шрифт з підсвіткою

### Анімації
- Використовувати `v-click` для поетапного розкриття
- Transitions: `slide-left`, `fade-out` (subtle)
- Не перевантажувати анімаціями

## Live Demo

### Підготовка
- Мати fallback скріншоти для кожного demo
- Тестувати Monaco Editor блоки локально
- Підготувати відео-бекапи

### Інструменти для демо
- Claude Code CLI
- Cursor IDE
- VS Code з розширеннями

### Fallback план
1. Скріншот результату
2. Відео запис
3. Код в Monaco Editor (без виконання)

## Мова контенту

### Українською
- Заголовки слайдів
- Пояснення та описи
- Bullet points
- Presenter notes

### Англійською (можна залишати)
- Технічні терміни (Rules, Commands, Skills, MCP)
- Назви інструментів (Claude Code, Cursor)
- Код та конфігурації
- Назви файлів (.cursorrules, CLAUDE.md)

## Інтерактивні елементи

### QR-коди
- Репозиторій з прикладами
- Документація
- Контакти спікера

### Посилання
- Додавати в presenter notes
- Включати в останній слайд "Ресурси"

## Тайминг

### Перевірка часу
- 1 слайд ≈ 1-1.5 хв (без коду)
- 1 слайд з кодом ≈ 2-3 хв
- Live demo ≈ 3-5 хв кожне

### Буфер
- Залишити 2-3 хв на Q&A
- Мати "опціональні" слайди для скорочення
